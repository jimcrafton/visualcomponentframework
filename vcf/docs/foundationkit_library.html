FoundationKit library

Functional Specification

Introduction

top

The FoundationKit provides all of the core services for the rest of the VCF. This includes the following:

  • provides basic OS functionality such as:
    • Threads, Mutexes, Semaphore - provides support for multiple threads of execution and for controlling access to resources through synchronization primitives. Allows for several different ways to use threads in your C++ classes.
    • Files - provides basic file support like creating, copying, renaming etc.
    • Streaming IO - allows for flexible streaming mechanism that is simple and intuitive to use. A variety of streams are implemented for you such as file streams, memory streams, text streams, etc.
    • Registry - a central database that allows for application settings to be stored and queried.
    • Libraries - allows for dynamically loading DLLs (or SOs) and invoking exported functions that are exposed by the library.
    • Process - allows for creating a process and querying for information such as the process ID and thread ID and process name.
  • provides RTTI, Class, Property, etc
  • Support Error logging - making it easy to store errors or messages to a log file.
  • XML parsing - supports basic XML parsing, with a non validating parser. 
  • Exceptions - provides a nice set of exceptions, which can also output their messages to the Error log if one is in use.
  • Memory tracking in debug mode - allows outputting any unreleased memory to the output buffer making it easier to track down memory leaks.

When using the FoundationKit you must initialize it  (see Detailed Description below) since there are a number of objects that need to be created. This is accomplished using the FoundationKit::init() static method call. Likewise when your application shuts down it needs to tell the FoundationKit to clean up, which is accomplished with the FoundationKit::terminate() static method call.

Detailed Description

top

The FoundationKit provides a wide variety of services and classes. Most of these are useful in applications, particularly console applications ( applications that require a GUI may use these as well, but have additional requirements that call for using the ApplicationKit instead ). 

When using the FoundationKit in your application you will need to initialize it with the call to FoundationKit::init() which starts up the basic services and creates the various singletons used in the FoundationKit . These singletons include the following objects:

  • The VCF::SystemToolkit - this singleton provides access to the various peer classes that represent things like a process or thread or file. These peer's implementation are OS specific. The VCF classes that wrap this functionality, such as the VCF::Process class which contains a VCF::ProcessPeer, must get their peers by talking to the VCF::SystemToolkit.
  • The VCF::System - this singleton provides access to low level OS functions like sleep, print, and error print.
  • The VCF::ClassRegistry - 

In addition there a few basic classes that are "registered" into the VCF::ClassRegistry at this time. These include:

  • VCF::Object
  • VCF::ObjectWithEvents
  • VCF::Rect

Introduction

top

The FoundationKit is the core library of the whole VCF. It must be linked to and then initialized and un-initialized when your application is finished with it. It is appropriate to use in a wide variety of applications, from console applications to GUI's.

Basic Usage

top

The FoundationKit is initialized by calling the static method  FoundationKit::init() - this should happen immediately after your application starts. 

The FoundationKit is terminated by a call to the static method FoundationKit::terminate() just prior to you applications closing.

Examples

top

 

//example demonstrating how to initialize and un-initialize the FoundationKit
//this presumes that the vcf/include directory is somewhere reachable in your compiler's path

#include "FoundationKit.h"

int main( int argc, char** argv )
{
	//initialize the FoundationKit
	VCF::FoundationKit::init();
	
	//your application code
	VCF::System::print( "Hello World!" );
	//more code here...
	
	//terminate the FoundationKit and release any resources it uses
	VCF::FoundationKit::terminate();

	return 0;
}
