No conflicts between vc6 and vc7 (.NET)

It is possible now to compile the vcf files with both vc6 and vc7 without any conflicts.

The subdirectory [debug] and [release] and similar ones, where the subproducts of the compilation are located, are now located under the directories [vc6] and [vc7] of each project.

For the sake of clarity, the libraries are located under the [vc60] and [vc70] subdirectories of the [build] directory.

The primary output have now the additional suffix _vc6 or _vc7.
For example bin/ApplicationKit_d.dll takes now the name bin/ApplicationKit_vc6_d.dll if compiled under vc6

Automatic library selection

The libraries are automatically selected by using two macros BUILD_APPKIT_LIB and USE_APPKIT_DLL
BUILD_APPKIT_LIB is only used when building the library (static or dynamic)
USE_APPKIT_DLL is only used when using the dynamic version of the library
