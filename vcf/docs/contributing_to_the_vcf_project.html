Contributing the VCF project

Introduction

The VCF Project needs help, so how about considering participating in the project ? Some of the things that still need doing:

  • Writing Components that extend the core functionality of the system
  • Documentation
  • Testing, and writing test/tutorial apps
  • Integrating libart into the GraphicsKit
  • Integrating the embedded mozilla browser. This is started, but needs more work...
  • Begin the port to MacOSX and/or Linux
  • Getting the framework to compile with Borland C++ compiler, and/or GCC 
  • Expanding on the work already done in the RemoteObjectKit, for use in distributed systems.
  • Helping develop the VCFBuilder

Sound interesting ? Well here's the list of what you'll need to do :

  • Register as a user at SF
  • Set up CVS
  • Read the Documentation Standards
  • Read the Naming standards
  • Read how to work with CVS
