Build Steps

Building the full VCF requires an XML parser used in the reading of Visual Form Files (VFF). Currently I am using the Xerces XML Pareser from Apache. It can downloaded at the Xerces site, or click below for a zip file of just the headers, libs, and dlls you need for the VCF. Please make sure it is installed in the following directory relative to the VCF root

root

  |
  +-- vcf
  |

  +--include
  |
  |... rest of VCF directories
  |

  |

  | ....rest of directories under include

Building

Under Win32 with VC++ 6.x

The VCF is built using the following settings (these are the exceptions to the standard settings for DLL projects):

  • C++ RTTI turned on

  • Code generation is Multi threaded DLL

  • Pre compiled headers are turned off generally, though I am experimenting with using them on my current build (as of 12/21/2000).

  • The following defines are neccessary:

    • FoundationKit : WIN32,_DEBUG,_WINDOWS,_MBCS,_USRDLL,FRAMEWORK_DLL,FRAMEWORK_EXPORTS,NO_MFC

    • GraphicsKit:WIN32,_DEBUG,_WINDOWS,_MBCS,_USRDLL,FRAMEWORK_DLL,GRAPHICSKIT_DLL,GRAPHICSKIT_EXPORTS

    • ApplicationKit: WIN32,_DEBUG,_WINDOWS,_MBCS,_USRDLL,FRAMEWORK_DLL,GRAPHICSKIT_DLL,APPKIT_DLL,APPKIT_EXPORTS

  • The following includes are required:

  • the libraries the VCF links with are (the [] represent optional names depending on whether the build is a debug or release build):

    • FoundationKit links with Rpcrt4.lib

    • GraphicsKit links with FoundationKitDLL[_d].lib, opengl32.lib, glu32.lib, and glaux.lib

    • ApplicationKit links with FoundationKitDLL[_d].lib, GraphicsKitDLL[_d].lib, xerces-c_1[D].lib, comctl32.lib, rpcrt4.lib, opengl32.lib, glu32.lib, and glaux.lib

    • NetworkKit links with FoundationKitDLL[_d].lib and Ws2_32.lib

 

When building from scratch, the order of building the various DLL's is as follows: FoundationKitDLL first, then the GraphicsKitDLL and then the ApplicationKitDLL.
