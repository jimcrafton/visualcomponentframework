Build Steps

 

Building

Under Win32 with VC++ 6.x

The VCF is built using the following settings (these are the exceptions to the standard settings for DLL projects):

  • C++ RTTI turned on

  • Code generation is Multi threaded DLL

  • Pre compiled headers are turned on generally, at least for the core projects, FoundationKit, ApplicationKit, and GraphicsKit.

  • The following defines are neccessary:

    • FoundationKit : WIN32,_DEBUG,_WINDOWS,_MBCS,_USRDLL,FRAMEWORK_DLL,FRAMEWORK_EXPORTS

    • GraphicsKit:WIN32,_DEBUG,_WINDOWS,_MBCS,_USRDLL,FRAMEWORK_DLL,GRAPHICSKIT_DLL,GRAPHICSKIT_EXPORTS

    • ApplicationKit: WIN32,_DEBUG,_WINDOWS,_MBCS,_USRDLL,FRAMEWORK_DLL,GRAPHICSKIT_DLL,APPKIT_DLL,APPKIT_EXPORTS

  • The following includes are required:

  • the libraries the VCF links with are (the [] represent optional names depending on whether the build is a debug or release build):

    • FoundationKit links with Rpcrt4.lib

    • GraphicsKit links with FoundationKit[_d].lib, opengl32.lib, glu32.lib, and glaux.lib

    • ApplicationKit links with FoundationKit[_d].lib, GraphicsKit[_d].lib, comctl32.lib, rpcrt4.lib, opengl32.lib, glu32.lib, and glaux.lib

    • NetworkKit links with FoundationKit[_d].lib and Ws2_32.lib

 

When building from scratch, the order of building the various DLL's is as follows: FoundationKit first, then the GraphicsKit and then the ApplicationKit.
