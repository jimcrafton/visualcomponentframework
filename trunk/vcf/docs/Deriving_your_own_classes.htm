Deriving your own classes

Most classes in the VCF can be safely derived from. Often you'll find all you need is to derive from VCF::Object. If you think your class will need to support the firing of events and the ability of allowing other objects to respond to these events, then deriving from VCF::ObjectWithEvents may be more appropriate. In terms of size difference, VCF::Object requires 8 bytes of memory, while VCF::ObjectWithEvents weighs in at 48 bytes. There are a variety of reasons for this, almost all relating to event handling. For more information on this please see Events.

Classes that need to be visually displayed, such as controls, should derive from the appropriate control class. For creating custom controls from scratch a good starting point is the VCF::CustomControl class. For more information on creating new components or controls please see the Components section.

When creating custom event classes you will probably want to derive from the VCF::Event class.
